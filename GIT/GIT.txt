gitk - интерфейс (гиткей)
Репозиторий - это хранилище всех версий кода (локальных, удаленных)
git init - Инициализация нового репозитория (создает доп. файлы)(иниц. нужно находясь в каталоге)
git clone (ссылка).git- (клонирует проект)

git add (. индексировать файл и которые в папках)(.* индексировать файл и которые только в одной папке)

git commit -m ""
commit — это снимок ваших локальных файлов, записанный в локальный репозиторий. 
Вопреки распространённому мнению, git хранит не только изменения между файлами, 
но и в целом полную версию всех файлов.
ключ -m - краткое описание
git commit -a -m "aded commit" - если файл ранее был заиндексирован (add) можно, не прописывать снова (add)

Git log - история commit
Хеш коммита (хэш коммита, commit hash) — это специальная метка, 
позволяющая отличать одни коммиты от других. 

git checkout (Хеш коммита)- откат назад, переключение по веткам.
git checkout -b newbranch - переключение и создание ветки с названием "newbranch"

git reset - отмена индексации
git revert - отмена коммита

git branch -a = проверить все ветки
git branch - проверка текущей ветки (*)
git branch nazva - создание новой ветки
git branch -d - удаление ветки (локально)

git merge - слияние веток (последовательные коммиты)
git rebase - слияние веток (не последовательные коммиты)

git push origin main = загрузить на удаленный репозиторий
git push origin --delete = удаление ветки (та которая глобальна (на репозитории))
git pull origin main = выгрузить с удаленного репозитория
